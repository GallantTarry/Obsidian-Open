好勒，少侠！扔掉那个鼠标经常卡住、复制粘贴还不方便的 VMware 图形窗口，是你迈向纯粹后端开发环境的第一步。

咱们直接开干，只需要在刚装好的 Ubuntu 里敲几行基础命令，就能把这座“桥”搭好：

### 1. 唤醒终端与更新软件列表

在你的 Ubuntu 桌面环境里，直接按下快捷键 **`Ctrl + Alt + T`**，调出那个黑底白字的终端窗口。

因为是刚出炉的新系统，先让它去获取一下最新的软件列表，输入并回车：

Bash

```
sudo apt update
```

_提示：输入 `sudo` 开头的命令意味着你要借用最高权限，系统会要求你验证密码。在 Linux 终端里输密码时，**屏幕上绝对不会显示任何星号或字符**，这是正常的防偷窥机制，大胆盲打完按回车就行。_

### 2. 安装并启动 SSH 服务端

获取到最新列表后，接下来就把 OpenSSH 的服务端装进去：

Bash

```
sudo apt install openssh-server -y
```

装完之后，Ubuntu 默认会自动把 SSH 服务拉起来。为了确认它是不是在后台踏踏实实地干活，咱们查一下它的状态：

Bash

```
sudo systemctl status ssh
```

如果你看到有一行亮着绿色的 `active (running)`，这就说明服务已经完美运行了。此时按下键盘上的 **`q`** 键退出状态面板。

### 3. 揪出虚拟机的 IP 地址

现在咱们得知道物理机该往哪个地址去连，输入命令查看网络信息：

Bash

```
ip a
```

在输出的一长串信息中，跳过那个名为 `lo`（本地回环地址）的网卡，往下看，找一个类似 `ens33` 或 `eth0` 的主网卡。在它下面会有一行写着 `inet 192.168.x.x/24` 的字样，这个 **`192.168.x.x`** 就是你这台虚拟机的局域网 IP 地址，把它记下来。

### 4. 物理机直连，完成“夺舍”

最爽的一步来了。现在，你可以直接把 VMware 那个巨大的窗口最小化，让它安安静静在后台跑着就行。

回到你熟悉的 Windows 物理机中，打开你顺手的终端工具（无论是 Windows 自带的 CMD、PowerShell，还是以后可能会用到的 MobaXterm、Xshell 都可以）。

在 Windows 的命令行里敲下：

Bash

```
ssh 你的Ubuntu用户名@刚才查到的IP地址
```

这个用户名是tukuai




_(例如：`ssh ubuntu@192.168.1.100`)_

敲下回车后，系统第一次连接会弹出一句长长的英文，问你是否确认连接并信任这台陌生的主机（Are you sure you want to continue connecting）。直接敲下 **`yes`** 并回车，然后盲打输入你的 Ubuntu 密码。

按下回车的瞬间，当你的 Windows 终端前缀变成了类似 `用户名@虚拟机名:~$` 的提示符时，这台虚拟机的远程操控权就彻底被你拿下了。所有的网络排查、代码部署、底层测试，从此都可以在这行极简的命令行里流畅完成。
