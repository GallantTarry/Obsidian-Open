来，少侠，我们继续用武侠角色作为例子。

方法重写（Method Overriding）是面向对象多态性的核心。简单来说，就是**子类觉得父类的方法不够酷、不够强，于是用完全相同的方法签名，自己重新写了一套更厉害的实现。**

这里有一个直观的代码实现：

Java

```
// 1. 父类：普通英雄
class Hero {
    protected String name;

    public Hero(String name) {
        this.name = name;
    }

    // 父类的方法：普通的物理攻击
    public void attack() {
        System.out.println(name + " 发动了普通攻击，造成了基础物理伤害。");
    }
}

// 2. 子类：剑客（继承自 Hero）
class Swordsman extends Hero {
    private String swordName;

    public Swordsman(String name, String swordName) {
        super(name); // 调用父类的构造器初始化 name
        this.swordName = swordName;
    }

    // 【核心：重写父类的方法】
    // 加上 @Override 注解，让编译器帮你严格把关（方法名、参数、返回值必须与父类一致）
    @Override
    public void attack() {
        // 如果想保留父类的动作，可以用 super.attack() 触发，这里我们直接完全覆盖
        System.out.println(name + " 拔出【" + swordName + "】，使出了一记「横扫千军」！漫天剑气激荡！");
    }

    // 子类特有的方法
    public void parry() {
        System.out.println(name + " 举起 " + swordName + " 进行了格挡，免受 50% 伤害。");
    }
}

// 3. 运行测试
public class GameTest {
    public static void main(String[] args) {
        System.out.println("=== 场景一：父类实例化 ===");
        Hero soldier = new Hero("路人甲");
        soldier.attack(); // 触发父类的普通攻击

        System.out.println("\n=== 场景二：子类实例化 ===");
        Swordsman lixiaoyao = new Swordsman("李逍遥", "无尘剑");
        lixiaoyao.attack(); // 触发子类重写后的炫酷攻击
        lixiaoyao.parry();  // 触发子类独有的格挡技能

        System.out.println("\n=== 场景三：多态（父类引用指向子类对象） ===");
        // 运行时绑定（Dynamic Binding）：编译时认为是 Hero，但运行时实际执行 Swordsman 的重写方法
        Hero gainie = new Swordsman("盖聂", "渊虹");
        gainie.attack(); // 依然会触发子类重写的「横扫千军」！
        
        // gainie.parry(); // 注意：编译报错！父类引用无法直接调用子类特有的方法
    }
}
```

## 重写（Override）的硬性江湖规矩

在 Java 中，重写必须要遵守非常严格的约束，业界通常总结为 **“两同、两小、一大”**：

- **两同（完全一致）**：
    
    - **方法名**必须相同。
        
    - **参数列表**必须完全一致。
        
- **两小（更加安全）**：
    
    - **返回值类型**：子类重写方法的返回值类型必须与父类一致，或者是父类返回值类型的子类（协变返回类型）。
        
    - **异常抛出**：子类重写方法声明抛出的异常范围，不能大于父类抛出的异常。
        
- **一大（权限更宽）**：
    
    - **访问权限**：子类的访问权限**不能比父类更窄**。例如，如果父类方法是 `protected`，子类重写时只能是 `protected` 或 `public`，绝对不能降级为 `private` 或默认包访问权限。
        

> ⚠️ **避坑指南：** 被 `private`、`static` 或 `final` 修饰的方法是**不能被重写**的。
