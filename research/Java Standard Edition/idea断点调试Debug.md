其实，调试（Debug）的本质非常浪漫——**它就像是你掌握了让代码世界“时间静止”的魔法**。你可以让程序在你指定的地方停下来，然后你一步一步地推着它往前走，亲眼看着每一个变量在这一瞬间变成了什么样子。

既然你提到了 IDEA（通常指 IntelliJ IDEA，Java 开发的神器，它和 CLion 的调试逻辑是完全互通的），我们就用一段带有逻辑缺陷的 Java 代码来练手。

### 1. 你的“靶子”：一段有 Bug 的代码

假设你现在正在写一个帮公司计算“前5个月招标项目总金额”的方法。代码如下，你可以把它复制到你的 IDEA 里：

Java

```
public class DebugPractice {
    public static void main(String[] args) {
        // 假设这是前5个月每个月的招标金额（万元）
        int[] monthlyAmounts = {10, 20, 30, 40, 50};
        
        int total = calculateTotal(monthlyAmounts);
        
        // 我们期望的总和是 10+20+30+40+50 = 150
        System.out.println("期望总金额: 150 万元");
        System.out.println("实际计算金额: " + total + " 万元");
    }

    // 计算总金额的方法
    public static int calculateTotal(int[] amounts) {
        int sum = 0;
        // 注意看这里的循环
        for (int i = 1; i < amounts.length; i++) {
            sum += amounts[i];
        }
        return sum;
    }
}
```

_如果你直接运行这段代码，会发现实际计算金额是 140，少了 10。_ 作为一个老手，你可能一眼看出了问题，但在复杂的业务代码里，这种逻辑错误是非常隐蔽的。这时候就需要 Debug 上场了。

### 2. IDEA 调试的“三大核心心法”

在 IDEA 里调试，你只需要掌握这三个动作：

- **第一式：设下埋伏（打断点）**
    
    在你觉得可能有问题的代码行号旁边（比如 `int sum = 0;` 这一行的最左侧数字边缘），用鼠标**左键点击一下**。会出现一个**红色的圆点**，这就叫断点（Breakpoint）。
    
    - _操作：点击 `calculateTotal` 方法里的 `int sum = 0;` 这一行左侧。_
        
    - _启动：不要点绿色的运行三角，而是点旁边那个**像小虫子一样的绿色按钮 (Debug)**，或者按 `Shift + F9`。_
        
- **第二式：步步为营（Step Over / F8）**
    
    当程序运行到红点处时，时间就静止了，这一行会高亮显示（通常是蓝色背景）。此时代码**还没执行**这一行。
    
    按下快捷键 **`F8`**（或者点击调试面板里那个像“折线箭头跨过一个点”的图标）。程序就会执行完当前行，停在下一行。
    
    - _秘诀：每按一次 F8，你都要盯着 IDEA 下方的 **Variables（变量）面板**，看里面的数字发生了什么变化。_
        
- **第三式：顺藤摸瓜（Step Into / F7）**
    
    如果你的代码调用了另一个复杂的方法，你想钻进那个方法内部去看看，就按 **`F7`**（向下指的箭头）。它会带你进入方法内部。
