少侠，这是一个非常深入且容易让人混淆的知识点。很多人在初学时，会被“对象作为参数传递时看起来像引用传递”所迷惑。

简单一句话：**Java 永远只有值传递 (Pass-by-Value)。**

为了让你彻底看清这背后的机制，我们需要区分什么是“值”，以及“值”到底是什么。

### 1. 什么是值传递？

值传递的核心在于：**无论你传递的是什么，Java 都会在传递过程中拷贝一份数据的副本。**

- 如果传递的是**基本数据类型**（如 `int`, `double`）：传递的是这个数值本身（比如 `10` 或 `3.14`）。
    
- 如果传递的是**引用数据类型**（如 `Object`, `Array`）：传递的是这个对象在堆内存中的**内存地址值**。
    

也就是说，不管是数字还是地址，Java 统统把它们当成“值”来复制一份。

### 2. 核心区别：发生了什么？

我们将两种情况放在一起看，你会发现逻辑是一样的。

#### 情况 A：基本类型（修改的是副本）

Java

```
public void changeInt(int x) {
    x = 100; // 修改的是本地变量 x 的值，原数据不会变
}
// 调用：int a = 10; changeInt(a); // a 依然是 10
```

#### 情况 B：引用类型（修改的是地址指向的对象）

这也是最让少侠困惑的地方。请看代码：

Java

```
public void changeObject(User user) {
    user.setName("少侠"); // 这里通过地址修改了对象内部属性
}
```

**这里发生了什么？**

Java 并没有把 `user` 对象本身传进去，而是把 `user` 变量里存的那个“内存地址值”拷贝了一份传给方法。

- 虽然地址是拷贝的（值传递），但**两个地址都指向了堆内存中的同一个对象**。
    
- 所以，你在方法里修改属性，外面的对象看起来变了，但这**不是因为引用传递**，而是因为你通过拷贝来的“地址值”找到了同一个本体进行修改。
    

### 3. 决定性的测试（证据）

如果你还不信，试试看在方法里直接给对象变量赋新值：

Java

```
public void test(User user) {
    // 如果是引用传递，这里会修改外面的引用指向
    // 但因为是值传递（拷贝了地址），这里只修改了本地的副本指向
    user = new User("新人"); 
}
```

当你调用这个方法后，你会发现外面的 `user` 依然指向原来的对象。**这证明了传入的确实只是一个地址的副本，而不是引用本身。**

### 4. 总结与对比（图解逻辑）

- **基本类型值传递**：传递的是数据的**数值副本**。
    
- **引用类型值传递**：传递的是对象**引用地址的数值副本**。
    

少侠，你可以这样理解：

- **基本类型**：你给别人一张写着“10元”的纸条，别人改了纸条上的数字，你手里的钱（原变量）不会变。
    
- **对象类型**：你给别人一张写着“地址”的纸条，别人按地址找到你家，把你家里的家具（对象属性）换了，你回家后发现家具变了。但这并不代表你把“家”本身传给别人了，你只是传了一份“住址”的拷贝。
