

这个问题非常经典，很多刚接触 Maven 的人都会有这个疑惑。少侠，实际上，下载的依赖并不是直接放到你项目的 lib 目录里的。

早期我们写 Java 项目时，确实是手动去网上下载 .jar 包，然后复制粘贴到项目的 lib 文件夹里。但 Maven 彻底改变了这种做法。以下是它的真正运作方式：

### 1. 依赖真正的藏身之处：本地仓库 (Local Repository)

当你在 pom.xml 中添加了一个依赖，Maven 会去网上的服务器（中央仓库）把它下载下来，然后存放在你电脑里的一个集中式大仓库中，这被称为本地仓库。

- Windows 默认路径： C:\Users\你的用户名\.m2\repository
    
- Mac/Linux 默认路径： ~/.m2/repository
    

为什么这样做？ 为了复用。如果你有 10 个项目都用到了 spring-web.jar，Maven 只需要在电脑里存一份就够了，不需要在每个项目的 lib 里都塞一份，极大地节省了硬盘空间。

### 2. 既然不在项目中，代码是怎么运行的？

这要分“写代码”和“打包上线”两个阶段来看：

#### 开发阶段（在 IDEA 等工具中运行）

当你在写代码时，IDEA 会自动读取你的 pom.xml 文件，然后去你的本地仓库里找到对应的 .jar 包。接着，IDEA 会在后台通过配置环境变量（Classpath），把你电脑里的这个 .jar 包“链接”到当前的项目中。

- 结果： 你的代码能正常调用这些依赖里的类，但你的项目文件夹里并没有产生任何真实的 .jar 文件。
    

#### 打包阶段（准备部署上线）

当项目写完了，你要把它变成一个可以到处运行的程序时，这些依赖到底去哪了，取决于你的项目类型和打包插件：

- 普通 Java 项目： 默认情况下，Maven 打包出来的 .jar 文件只包含你自己写的代码。如果你想把依赖拿出来放到 lib 文件夹，或者把它们合并到一个大文件里，你需要配置特定的 Maven 插件（如 maven-dependency-plugin 或 maven-shade-plugin）。
    
- Spring Boot 项目： Spring Boot 有一个极其强大的打包插件。当你执行打包操作时，它会把你自己的代码以及本地仓库里所有用到的依赖 .jar 包，全部打包进一个超级大的 .jar 文件（俗称 Fat JAR）内部的 BOOT-INF/lib 目录里。
    
- 传统 Web 项目 (.war)： Maven 打包时，会自动把你需要的依赖 .jar 包从本地仓库复制出来，塞进生成的 .war 文件内部的 WEB-INF/lib 文件夹下。
    

总结来说： 你不用再去管理任何的 lib 文件夹了。平时开发时，所有依赖都在电脑的统一存放点（本地仓库）；最终打包时，Maven 的插件会自动帮你把需要的依赖塞进最终的发布包里。
