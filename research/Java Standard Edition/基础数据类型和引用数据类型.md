少侠，Java 中的数据类型体系非常清晰，主要分为**基本数据类型**（Primitive Types）和**引用数据类型**（Reference Types）两大阵营。下面为你详细梳理：

### 一、 基本数据类型 (Primitive Types)

Java 中一共有 **8 种**基本数据类型，它们是语言底层内置的，直接存储具体的数据值。可以分为四大类：

**1. 整数类型**

- **`byte`**: 8 位（1 字节），取值范围 -128 到 127。
    
- **`short`**: 16 位（2 字节），取值范围 -32,768 到 32,767。
    
- **`int`**: 32 位（4 字节），开发中最常用的整数类型。
    
- **`long`**: 64 位（8 字节），用于表示极大的整数。赋值时通常在末尾加 `L`（例如 `1000L`）。
    

**2. 浮点数类型（小数）**

- **`float`**: 32 位（4 字节）单精度浮点数，赋值时末尾必须加 `F` 或 `f`（例如 `3.14f`）。
    
- **`double`**: 64 位（8 字节）双精度浮点数，精度更高，是 Java 中默认的小数类型。
    

**3. 字符类型**

- **`char`**: 16 位（2 字节）的 Unicode 字符，只能存储单个字符，且必须用单引号括起来（例如 `'A'`、`'中'`）。
    

**4. 布尔类型**

- **`boolean`**: 只有两个值：`true` 或 `false`。常用于逻辑控制和条件判断。
    

### 二、 引用数据类型 (Reference Types)

在 Java 中，**只要不是上述的 8 种基本类型，其他的全都是引用数据类型**。引用类型本身不直接存储数据的内容，而是存储指向内存中某个对象的**地址（引用）**。

主要包含以下几大类：

- **类 (Class)**:
    
    - Java 标准库提供的类，如极其常用的字符串 `String`、根类 `Object` 等。
        
    - 基本数据类型对应的**包装类**（Wrapper Classes），如 `Integer`、`Double`、`Character` 等。
        
    - 开发者自己写的任何自定义类（比如 `User`、`Car`、`HavenPanel` 等）。
        
- **接口 (Interface)**: 比如集合框架中的 `List`、`Map`，或者线程相关的 `Runnable`。
    
- **数组 (Array)**: 无论数组内部装的是基本类型还是引用类型，**数组本身统统属于引用类型**（例如 `int[]`、`String[]`、`Object[]`）。
    
- **枚举 (Enum)**: 用于定义一组特定的、固定的常量集合。
    
- **注解 (Annotation)**: 代码中的元数据标签，比如大家熟知的 `@Override`。
    

### 三、 两者的核心区别

为了更直观地理解，可以通过以下几个维度进行对比：

|**特性**|**基本数据类型**|**引用数据类型**|
|---|---|---|
|**内存位置**|数据值直接存放在**栈 (Stack)** 中，读取速度极快。|引用（内存地址）存在栈中，而实际的对象内容存放在**堆 (Heap)** 中。|
|**默认值**|声明后如果不赋值，有自带的默认值（如 `int` 是 `0`，`boolean` 是 `false`）。|声明后如果不初始化，默认值永远是 **`null`**。|
|**功能特性**|只是纯粹的数据载体，没有任何内部方法或属性。|是完整的对象，可以调用类中定义的各种方法和属性。|
|**内存大小**|大小是绝对固定的（例如 `int` 永远占用 4 个字节）。|大小是不固定的，取决于内部对象实际分配了多少数据。|
