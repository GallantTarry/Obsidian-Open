少侠，这是一份关于“堆（Heap）”这一数据结构的详细教学。

## 核心要点概括

- **本质**：堆是一种特殊的完全二叉树。
    
- **分类**：分为最大堆（父节点的值大于或等于子节点）和最小堆（父节点的值小于或等于子节点）。
    
- **物理存储**：通常使用数组（Array）来连续存储，无需指针，空间利用率极高。
    
- **时间复杂度**：获取堆顶极值的时间复杂度为 $O(1)$，插入节点和删除节点的时间复杂度均为 $O(\log n)$。
    
- **主要应用场景**：优先队列（Priority Queue）、堆排序（Heap Sort）、求解 Top K 极值问题。
    

## 一、 堆的逻辑结构与物理存储

堆在逻辑层面是一棵完全二叉树，但在物理内存中，它是通过一维数组来实现的。以下是一个最大堆的静态结构图：

代码段

```
graph TD
    A((100)) --> B((19))
    A --> C((36))
    B --> D((17))
    B --> E((3))
    C --> F((25))
    C --> G((1))
```

在数组中，上述堆的元素按层序遍历的顺序存储为：`[100, 19, 36, 17, 3, 25, 1]`。

利用数组的索引，我们可以通过数学公式直接定位任意节点的父子关系。假设当前节点的索引为 $i$（索引从 0 开始计算）：

- 左子节点的索引为 $2i + 1$
    
- 右子节点的索引为 $2i + 2$
    
- 父节点的索引为 $\lfloor(i - 1) / 2\rfloor$
    

## 二、 堆的核心操作机制

### 1. 插入新节点 (Insert)

- **初始放置**：将新元素直接添加到数组的末尾（即作为完全二叉树的最后一个叶子节点）。
    
- **上浮调整（Shift Up）**：将该节点与其父节点进行比较。如果违反了堆的性质（例如在最大堆中，新节点大于父节点），则交换这两个节点的位置。
    
- **终止条件**：不断重复上浮过程，直到该节点满足堆性质（小于或等于其父节点），或者该节点已经到达了根节点（数组首位）。
    

### 2. 删除堆顶元素 (Extract Max/Min)

- **提取目标**：将堆顶元素（数组索引 0 的位置）取出，它一定是当前集合中的最大值或最小值。
    
- **填补空缺**：将堆结构中的最后一个元素（数组的末位）移动到堆顶位置，并将数组长度减一。
    
- **下沉调整（Shift Down）**：将新的堆顶元素与其左右子节点进行比较。在最大堆中，如果父节点小于子节点，则将父节点与**较大的那个子节点**进行交换。
    
- **终止条件**：不断重复下沉过程，直到该节点大于或等于其所有存在的子节点，或者它已经沉降为叶子节点。
    

### 3. 数组建堆 (Heapify)

- **目的**：将一个完全无序的普通数组就地转化为一个有效的堆结构。
    
- **执行方式**：从最后一个非叶子节点（索引为 $\lfloor(n/2) - 1\rfloor$）开始，逆序向上遍历直到根节点，对每一个节点执行“下沉调整（Shift Down）”。
    
- **效率优势**：这种整体建堆的时间复杂度为 $O(n)$，远快于通过循环依次执行插入操作的 $O(n \log n)$。
    

## 三、 堆的典型应用场景

- **优先队列**：操作系统或程序需要动态处理带有优先级的任务，堆能够稳定、高效地维护并随时输出当前最高优先级的任务。
    
- **堆排序**：一种时间复杂度稳定在 $O(n \log n)$ 的原址排序算法。将无序数组构建为最大堆后，不断将堆顶（最大值）与数组末尾元素交换，并缩小有效堆的范围重新下沉，最终可以得到一个升序数组。
    
- **海量数据 Top K 问题**：在 10 亿个数据中寻找最大的前 10 个数。只需在内存中维护一个容量为 10 的最小堆，遍历海量数据与最小堆的堆顶对比。遇到更大的值就替换堆顶并重新下沉调整，扫描结束后，堆内的 10 个元素即为目标答案，极大节省了内存开销。
