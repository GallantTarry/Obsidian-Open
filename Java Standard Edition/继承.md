少侠，我们在做独立游戏或者写工具箱的时候，经常会用 Java 的继承（`extends`）来搭框架。继承就像是“拿来主义”，但并不是父类的所有东西都能被原封不动地拿走。

简单来说，继承能继承的是“家产（属性）”**和**“手艺（方法）”**，但**“构造器（图纸）”**和**“私房钱（私有内容）”是继承不了的。

我们把继承的内容拆解开来看：

## 1. 能够被完美继承的（可以直接用）

只要父类没有刻意隐藏，子类就能顺理成章地接手：

- **公共的（`public`）**：父类所有的 `public` 属性和方法，子类都可以直接使用。
    
- **受保护的（`protected`）**：专门为继承设计的修饰符。子类（以及同一个包下的其他类）可以直接访问。
    
- **默认的（不写修饰符 / 包访问权限）**：如果子类和父类在同一个包（Package）里，那就能继承；如果不在同一个包，就继承不到。
    

## 2. “继承”了，但不能“直接碰”的（私房钱）

这是最容易让人产生错觉的地方：**父类的 `private`（私有）属性和方法。**

- **从内存机制上说**：当你 `new` 一个子类对象时，父类的 `private` 属性其实**已经包含**在这个对象所占用的内存空间里了。所以从物理层面上，它“继承”过来了。
    
- **从访问权限上说**：子类**绝对不能直接访问**（比如 `this.私有变量名` 是会报错的）。
    
- **如何使用？** 只能通过父类提供的 `public` 或 `protected` 方法（比如 `getter/setter`，或者父类的其他业务方法）来间接读取或修改这些私有数据。
    

> 💡 **游戏开发视角的比喻**：
> 
> 父类英雄有一个私有的 `心脏` 属性。子类剑客也有一颗一模一样的心脏，但是剑客不能自己把胸膛剖开直接修改心脏跳动的频率，只能调用父类提供的公开方法 `吃药()` 来恢复心脏的血量。

## 3. 绝对无法继承的（只有父类专属）

这两样东西，子类是彻底拿不到的：

1. **构造器（Constructors）**：
    
    - 构造器是用来构建对象特定图纸的。父类的图纸叫 `Hero()`，子类叫 `Swordsman()`，名字都不一样，没法继承。
        
    - 我们之前聊过，子类只能用 `super()` 去**调用**父类构造器来完成初始化，但不能把它占为己有。
        
2. **初始化块（Static Blocks / Instance Blocks）**：
    
    - 无论是静态代码块（`static {}`）还是普通代码块（`{}`），它们都是属于类或对象初始化的专属逻辑，不会被继承。不过，在实例化子类时，父类的这些代码块会优先于子类的代码块自动执行。
        

## 避坑指南：属性不谈“重写”，只谈“隐藏”

我们知道，父类的方法不够好，子类可以“重写（Override）”它。那父类的属性呢？

- **Java 中没有属性重写！** 如果子类定义了一个和父类名字完全相同的属性（哪怕类型不一样），这不叫重写，这叫**隐藏（Hiding）**。
    
- 这意味着，父类和子类的同名属性会**同时存在**于内存中。当你用 `this.属性名` 时，拿到的是子类的；当你用 `super.属性名` 时，拿到的是父类的。
