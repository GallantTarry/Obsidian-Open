Java 的数据结构主要集中在 **Java 集合框架（Collections Framework）** 中。这是 Java 开发中最核心的基础，不仅直接决定了代码的运行效率，也是技术进阶和面试中的重中之重。

Java 集合框架可以划分为大两阵营：单列集合 **`Collection`** 和双列键值对 **`Map`**。以下是必须掌握的核心知识点：

### 一、 List 体系（有序、可重复）

List 接口的特点是存取有序，且允许元素重复。

- **`ArrayList`（核心）**：
    
    - **底层**：基于动态数组实现。
        
    - **特性**：支持按索引快速随机访问，时间复杂度为 O(1)。但在数组中间插入或删除元素时，需要移动后续元素，时间复杂度为 O(N)。
        
    - **扩容机制**：默认初始容量为 10，当容量满时，会自动扩容为原来的 **1.5 倍**。
        
- **`LinkedList`**：
    
    - **底层**：基于双向链表实现。
        
    - **特性**：插入和删除元素速度极快（只需改变指针指向），时间复杂度为 O(1)。但不支持随机访问，查找元素必须从头或尾遍历，时间复杂度为 O(N)。
        
- **`Vector`**：
    
    - 老旧的线程安全集合，底层也是数组，通过 `synchronized` 保证线程安全，但性能极差，现代开发中已基本被淘汰。
        

### 二、 Set 体系（无序、不可重复）

Set 接口主要用于数据去重。

- **`HashSet`（核心）**：
    
    - **底层**：实质上就是一个由 `HashMap` 支持的包装类，数据存放在 HashMap 的 Key 中，Value 则是一个统一的默认对象。
        
    - **特性**：无序，操作速度快，时间复杂度为 O(1)。
        
- **`LinkedHashSet`**：
    
    - **底层**：在 `HashSet` 的基础上，额外维护了一个双向链表来记录元素的插入顺序。
        
    - **特性**：既保证了元素的唯一性，又能按照元素的插入顺序进行遍历。
        
- **`TreeSet`**：
    
    - **底层**：基于红黑树（`TreeMap`）实现。
        
    - **特性**：可以对存入的元素进行**自然排序**或**自定义规则排序**，时间复杂度为 O(log N)。
        

### 三、 Map 体系（Key-Value 键值对）

Map 用于存储映射关系，Key 必须唯一，Value 可以重复。

- **`HashMap`（重中之重）**：
    
    - **底层结构**：
        
        - JDK 1.7 及以前：数组 + 单向链表。
            
        - JDK 1.8 及以后：**数组 + 单向链表 + 红黑树**。当某一个数组槽位（Bucket）上的链表长度超过 **8** 且数组总长度超过 **64** 时，链表会转化为红黑树，将极端情况下的查询时间复杂度从 O(N) 优化到 O(log N)。
            
    - **扩容机制**：默认容量是 16，负载因子是 0.75。当存入的元素数量超过 `容量 * 0.75` 时，容量会翻倍。
        
    - **哈希冲突**：采用“拉链法”（链表）解决 Hash 碰撞。
        
- **`LinkedHashMap`**：
    
    - 在 HashMap 的基础上维护了双向链表，可以按插入顺序或访问顺序（常用于实现 LRU 缓存）遍历。
        
- **`TreeMap`**：
    
    - 基于红黑树实现，可以根据 Key 进行自动排序。
        
- **`ConcurrentHashMap`（并发核心）**：
    
    - **线程安全的 HashMap**。
        
    - JDK 1.8 中摒弃了原来的“分段锁（Segment）”，改用 **CAS 操作 + `synchronized`** 锁定当前节点（链表或红黑树的首节点），大大提升了并发度。
        

### 四、 Queue 与 Deque 体系（队列）

用于排队处理任务的场景。

- **`Queue`（单向队列）**：先进先出（FIFO）。
    
    - **`PriorityQueue`**：优先队列，底层基于“堆（Heap）”实现。它可以每次让优先级最高（或最小）的元素出队，常用于任务调度。
        
- **`Deque`（双端队列）**：允许在队列的两端进行插入和删除。
    
    - **`ArrayDeque`**：基于可变数组实现，作为栈（Stack）使用时比 `Stack` 类快，作为队列使用时比 `LinkedList` 快。
        

### 五、 核心拔高知识点（面试必问）

1. **Fail-Fast（快速失败）机制**：
    
    - 当使用迭代器（Iterator）遍历 `ArrayList` 或 `HashMap` 时，如果其他线程（或当前线程未通过迭代器本身）修改了集合的结构（增加或删除元素），会立即抛出 `ConcurrentModificationException` 异常。
        
2. **`equals()` 与 `hashCode()` 的契约**：
    
    - 在往 `HashSet` 或 `HashMap` 中存入自定义对象时，**必须重写这两个方法**。如果不重写，默认使用的是对象的内存地址，会导致逻辑上相同的对象被识别为不同的键，从而造成内存泄漏或数据丢失。
        
3. **时间复杂度对比**：
    
    - 需要随时能够在脑海中反应出各种操作的代价：`ArrayList` 的尾部追加是 O(1)，中间插入是 O(N)；`HashMap` 的查询正常是 O(1)，树化后是 O(log N)；`TreeMap` 的所有操作都是 O(log N)。
