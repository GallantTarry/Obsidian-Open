
因为你虽然没有敲 Maven 的打包命令，但 IDEA 的工件系统非常智能，它能够自动读取你项目中的 `pom.xml`。在你点击 `Build Artifacts` 的那一刻，主要会分为以下三种情况：

### 情况一：创建工件时，选了“提取到目标 JAR”（最常见）

如果你是通过 `File -> Project Structure -> Artifacts -> + -> JAR -> From modules with dependencies...` 创建的工件，IDEA 会弹出一个配置窗口。

如果你在 **JAR files from libraries** 选项中勾选了：

- **Extract to the target JAR（提取到目标 JAR）**
    

那么，**是的，所有的 pom 依赖都会被打入这个 JAR 包中**。IDEA 会在后台把 `pom.xml` 里声明的所有第三方 `.jar` 包全部解压，把里面的 `.class` 文件全掏出来，和你自己写的代码揉在一起，最终生成一个巨大的 **Fat JAR（胖包）**。

### 情况二：创建工件时，选了“复制到输出目录”

如果在上述同一个弹窗中，你勾选的是：

- **Copy to the output directory and link via manifest（复制到输出目录并建立清单链接）**
    

那么，pom 依赖**不会**进到你的 JAR 包内部。IDEA 会把你自己的代码打成一个干净的 JAR，然后把所有的 pom 依赖原封不动地复制到你输出目录的旁边（通常是一个 `lib` 文件夹），同时在你的 JAR 包内部的 `META-INF/MANIFEST.MF` 文件里写好类路径（Class-Path）。

### 情况三：创建的是“空工件”（Empty JAR）且没有手动拖拽

如果你在创建工件时，选择的是 `Artifacts -> + -> JAR -> Empty`（空工件）。那么默认情况下，这个工件**只包含你自己的模块编译输出（Compile Output）**。

在工件的配置界面中：

- 左侧是“输出结构（Output Layout）”，代表最终生成的 JAR 里包含什么。
    
- 右侧是“可用元素（Available Elements）”，里面会躺着一堆写着 `Maven: xxx:xxx` 的依赖库。
    

如果你**没有**把右侧的那些 Maven 依赖双击或者右键“加入（Add to Method）”到左侧的 JAR 结构里，那么这些 pom 依赖**绝对不会**被打进去。运行这个 JAR 时就会直接报 `ClassNotFoundException`。

### 💡 怎么验证你的 JAR 到底有没有包含依赖？

少侠可以用一个最简单直接的方法去验证：

1. 找到你打出来的 `.jar` 文件，把它的后缀名强行改成 `.zip`。
    
2. 解压这个压缩包，看看里面的目录结构：
    
    - 如果里面除了你自己的包名，还直接散落着很多其他框架的包名目录（比如 `org/yaml`、`com/alibaba` 等），说明依赖已经被**提取并合入**了（对应情况一）。
        
    - 如果里面只有你自己的包名和 `META-INF`，那说明依赖没有进包里（对应情况二或三）。
